Group Key Management for Multicast Communication (Final Design Project)

Multicast is a form of communication that delivers information from a source to a set of destinations simultaneously in an efficient manner; the messages are delivered over each link of the network only once and only duplicated at branch points, where the links to the destinations split. Important applications of multicast include distribution of financial data, billing records, software, newspapers, pay-per-view movies; audio/video conference; distance learning; and distributed online games. In order to ensure that only authorized users can access the multicast data, the data are encrypted using a cryptographic key known as the group key. The group key is known only to authenticated and authorized members of a multicast group. Every time a membership change occurs, the group key must be changed to ensure backward and forward secrecy. Group key management refers to the actions taken to update and distribute the group key upon members joining and leaving a multicast group. [Group Project]

C Process Monitoring Program: Unix



Synchronization of page usage for different processes was implemented in the get_process_mem(prc_id) function, this was the most challenging aspect of this project. Threads are created for each process passed in the watch command. The synchronization of each thread is maintained by global variables which keep track of each thread's status and clock_gettime() function keeps track of the time. Three different time operations are running at any given time for the watch command. These include the sleep() function, the clock_gettime() funciton and a time dependent integer variable. Therefore the server.c file is compiled with -lpthread library aswell as -lrt library to support clock_gettime() function. Once the log has finished recording all the page usage data and it sends it to the client then writes “*End of Watch *” to the socket. This is read by the client and it allows the client to exit the read() loop. [Individual Project]

The main objective of this project was to examine the efficiency of direct mapped cache controller and various aspects of the direct mapped cache that enable regulating memory access in an efficient manner.
